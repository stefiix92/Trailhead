# Governance

Trailhead is a small, maintainer-led project.

## Decision-making

- The maintainer(s) aim to be responsive and transparent in Issues/PRs.
- For ambiguous changes, we prefer writing down the “why” in the PR description (or a short doc) before merging.

## Scope discipline

Trailhead’s defining constraints are non-negotiable:

- **Logs-only** (no metrics/traces)
- **Query, don’t dump**
- **Evidence-first outputs** (line IDs and structured results)
- **Self-hosted** (no hosted/telemetry mode)
- **Read-only** (no write operations)

If a proposal violates these, it’s unlikely to be accepted — even if it’s useful.
