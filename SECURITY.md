# Security Policy

## Supported versions

Trailhead is early-stage. The `main` branch and the latest tagged release are considered supported.

## Reporting a vulnerability

Please **do not open a public issue** for suspected security vulnerabilities.

Instead, report privately:

- Email the maintainer(s) listed in the GitHub repository’s “Security” page (preferred), or
- If no contact is configured yet, open an issue with **no sensitive details** and ask for a private channel.

What to include:

- A clear description of the issue and impact
- Steps to reproduce
- A minimal proof-of-concept (if applicable)
- Any relevant logs/config (redacted)

## Disclosure process

We’ll acknowledge receipt, assess severity, and aim to ship a fix + advisory as quickly as practical.
